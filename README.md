# C++ Modules - From C to Object-Oriented Programming

This project consists of a series of modules designed to introduce **Object-Oriented Programming (OOP)** using **C++98**, focusing on low-level programming, manual memory management, and algorithm efficiency. Developed as part of the 42 School curriculum, these modules serve as a comprehensive journey from basic C++ syntax to advanced use of the Standard Template Library (STL).

## 📋 Project Overview

The curriculum is divided into 10 modules (00 to 09), each introducing new concepts step-by-step with increasing complexity.

### [Module 00: Basics](https://github.com/Mateus998/CPP-Modules/tree/main/CPP00)
Focuses on the transition from C to C++, introducing **namespaces, classes, member functions**, and **stdio streams**. Key exercises include a basic PhoneBook and recreating lost class logs.

### [Module 01: Memory & References](https://github.com/Mateus998/CPP-Modules/tree/main/CPP01)
Explores **memory allocation** (new/delete), **references**, and **pointers to members**. It covers the difference between stack and heap allocation and introduces the **switch statement** through the "Harl" automated complaint system.

### [Module 02: Operator Overloading](https://github.com/Mateus998/CPP-Modules/tree/main/CPP02)
Introduces **Ad-hoc polymorphism**, **operator overloading**, and the **Orthodox Canonical Class Form**. The main challenge is implementing a fixed-point number class to balance performance and precision.

### [Module 03: Inheritance](https://github.com/Mateus998/CPP-Modules/tree/main/CPP03)
Covers **inheritance** and the construction/destruction chaining of derived classes. It features the creation of various "Trap" robots (ClapTrap, ScavTrap, FragTrap) culminating in the multi-inherited DiamondTrap.

### [Module 04: Polymorphism & Interfaces](https://github.com/Mateus998/CPP-Modules/tree/main/CPP04)
Deep dives into **subtype polymorphism**, **abstract classes**, and **interfaces**. Exercises focus on deep copies (using a "Brain" class) and a complex Materia system for characters.

### [Module 05: Exceptions](https://github.com/Mateus998/CPP-Modules/tree/main/CPP05)
Introduces **repetition and exceptions**. It simulates a bureaucratic nightmare where Bureaucrats must sign and execute various Forms, handling errors through try/catch blocks.

### [Module 06: C++ Casts](https://github.com/Mateus998/CPP-Modules/tree/main/CPP06)
Explores the four C++ **casts** (static, dynamic, reinterpret, and const). Tasks include scalar type conversion, serialization of data structures, and identifying real types of objects without using typeinfo.

### [Module 07: Templates](https://github.com/Mateus998/CPP-Modules/tree/main/CPP07)
Introduces **generic programming** through **function templates and class templates**. Projects include generic swap/min/max functions and a custom Array template class.

### [Module 08: STL Containers](https://github.com/Mateus998/CPP-Modules/tree/main/CPP08)
Focuses on the **Standard Template Library (STL)**, specifically templated **containers, iterators, and algorithms**. It introduces `std::vector`, `std::list`, and the challenge of making a `std::stack` iterable (MutantStack).

### [Module 09: Advanced STL](https://github.com/Mateus998/CPP-Modules/tree/main/CPP09)
Final module dedicated to advanced STL usage. Key projects include a Bitcoin Exchange rate calculator, a Reverse Polish Notation (RPN) solver, and the **PmergeMe** sorter using the **Ford-Johnson algorithm**.

## 🧠 What I Learned

*   **Manual Memory Management:** Mastering `new` and `delete` to ensure **zero memory leaks**.
*   **The Orthodox Canonical Form:** Designing classes with a default constructor, copy constructor, assignment operator, and destructor.
*   **OOP Fundamentals:** Deep understanding of encapsulation, inheritance, and polymorphism.
*   **Error Handling:** Using exceptions to build robust and crash-resistant programs.
*   **Generic Programming:** Using templates to write flexible and reusable code.
*   **Data Structures:** Practical application of STL containers and complex sorting algorithms.

## ⚠️ Project Constraints

*   **Standard:** All code complies with **C++98**.
*   **Forbidden Functions:** Use of `*printf()`, `*alloc()`, and `free()` is strictly prohibited to ensure a "C++-ish" approach.
*   **Compilation Flags:** Every exercise is compiled with `-Wall -Wextra -Werror`.
*   **Strict Design:** Classes follow the Orthodox Canonical Form where required, and all headers use include guards.
